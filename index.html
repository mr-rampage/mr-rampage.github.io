DADOS 3.0 Checkpoint

Three Major Goals

  1. Data Integrity - data must be an accurate representation of the truth
  2. Stability - service and data must always be available
  3. Flexibility - system can support growth due to ever changing business needs

DADOS as a Platform

DADOS 3.0 will be designed as a platform constructed with two different tools:

  • Electronic Data Capture System - This component's goal is to allow the user to enter data into DADOS for storage.
  • Reporting System - This component's goal is to provide the user with analysis about the data stored within DADOS.

Both systems will appear under the same UI as DADOS 3.0, but from an architecture point of view will have separate applications and databases. This will allow for optimal performance.

Technical Decisions

To ensure data integrity, stability, and flexibility, the overall system design must be simple. This will be achieved through the following design decisions:

  1. No delete policy and automatic versioning of all data.
  2. Appropriate validation and error handling to ensure data integrity.
  3. Fail first development - system is designed to crash on any errors in development to ensure stability in production.
  4. Module based system to allow for flexible growth.
  5. Data Models structured using Generics to allow for change
  6. Security first development - ensure that data is safe to avoid manipulation of data
